# BOSS直聘职位搜索与分析 Skill

## 目标

把“浏览器自动化搜岗 → 抓取职位卡片 → AI 判断有效性 → 生成投递报告 → 飞书同步”固定成可复用流程，优先适配 **OpenClaw browser + `profile=user`**。

适用场景：
- 搜某岗位在某城市的招聘情况
- 过滤掉薪资/经验/活跃度明显不合适的岗位
- 总结值得投递的职位
- 辅助生成与 HR 的沟通建议

## 触发示例

- “帮我在 BOSS直聘 上搜索 AI 产品经理 的岗位”
- “找一下广州的 Python 开发 职位，薪资 15-25k”
- “BOSS直聘 搜一下 数据分析 实习”
- “给我同步到飞书表格，并附上投递建议”

---

## 运行前检查

### 1. 浏览器接入优先

执行前先确认：
1. `browser action="profiles"` 中存在 `user`
2. `browser action="status" profile="user" target="host"` 能正常连接
3. 若失败，不要反复重试 browser；先修复浏览器接入

### 2. Ubuntu + snap Chromium 环境说明

用户当前环境为 **Ubuntu + Chromium（snap 安装）**。
这类情况下，`profile=user` 常见问题是 OpenClaw 默认尝试连接 Google Chrome，而不是 Chromium。

优先处理顺序：
1. 确认 Gateway 正常运行
2. 明确设置 `browser.executablePath` 指向 Chromium
3. 如 `existing-session/chrome-mcp` 依然不稳定，改用 **浏览器扩展 Relay 模式**

### 3. 登录与风控

必须使用用户已登录的 BOSS 直聘浏览器会话。
若出现：
- 登录页
- 验证码
- 风控拦截
- 空白职位列表

则停止自动化，并让用户先手动完成验证。

## 参数提取

从用户输入中提取：
- `job_title`：职位名称，必填
- `city`：城市，默认按用户输入；无则提示补充或使用常见城市
- `salary_range`：可选，如 `15-25k`
- `experience`：可选，如 `1-3年`
- `education`：可选，如 `本科`
- `max_jobs`：默认 `10`，建议上限 `20`
- `generate_intro`：是否生成打招呼建议，默认 `false`
- `save_to_bitable`：是否写入飞书 Bitable，默认 `false`

## 执行流程

### 第一步：打开职位搜索页

优先直接构造搜索 URL，若失败再走页面交互。

示例：
`https://www.zhipin.com/web/geek/job?query=<job_title>`

城市、薪资、经验等筛选可在页面上进一步选择。

### 第二步：抓取职位列表

使用 `snapshot` 提取职位卡片，优先抓：
- 职位名称
- 公司名称
- 薪资范围
- 地点
- 经验要求
- 学历要求
- HR 活跃状态
- 可点击链接/ref

### 第三步：首轮筛选

优先过滤：
- 明显地点不符
- 薪资与用户目标明显偏离
- 经验要求差距过大
- HR 长期不活跃
- 描述极短、职责模糊、疑似挂岗

建议标签：
- ⭐ 优先：匹配度高、HR活跃、JD清晰
- ✅ 正常：基本匹配，可投
- ⚠️ 观望：部分条件一般，但仍可关注
- ❌ 过滤：不建议投递

### 第四步：进入详情页深挖

对保留下来的岗位，逐条进入详情页，提取：
- 岗位职责
- 任职要求
- 加分项
- 团队/业务线信息（若可见）
- 公司阶段/规模（若可见）

### 第五步：AI 判断与打分

每条岗位输出：
- `match_score`：0-10
- `salary_fit`：高 / 中 / 低
- `risk_flags`：如“要求过高”“描述模糊”“HR不活跃”
- `highlights`：1-3 条值得关注点
- `recommendation`：优先 / 可投 / 观望 / 放弃

### 第六步：汇总报告

建议格式：

```markdown
## BOSS直聘搜索报告：<job_title>（<city>）
搜索时间：YYYY-MM-DD HH:MM
分析岗位：Y
有效岗位：X

### ⭐ 优先岗位
1. <职位名称> - <公司名>
- 薪资：xxx
- 地点：xxx
- 要求：xxx
- HR活跃：xxx
- 推荐理由：...
- 链接：...

### ✅ 可投岗位
...

### ❌ 已过滤岗位
- <职位>：过滤原因

## 综合建议
- 当前市场画像
- 最值得优先投递的 3 个岗位
- 简历应强化的关键词
```

### 第七步：生成打招呼建议（可选）

若 `generate_intro=true`，为优先岗位生成 50-80 字开场话术：
- 点明自己最匹配的 1-2 点
- 避免空话套话
- 最后留一个轻问题，引导 HR 回复

只生成建议，不自动发送。

### 第八步：同步到飞书 Bitable（可选）

推荐字段：
- 关键词
- 平台
- 标题
- 作者/公司
- 是否广告（岗位场景可写“否”或留空）
- 综合评分
- 相关度/匹配度
- 核心观点/推荐理由
- 链接
- 搜索时间

尽量按批次写入，而不是高频逐条写入。

### 第九步：进度汇报

长流程中按阶段汇报：
- 已开始搜索
- 已抓到多少职位卡片
- 已完成多少条详情分析
- 是否遇到登录或风控
- 是否已生成总结/同步飞书

## 错误处理

| 情况 | 处理方式 |
|---|---|
| `browser` 超时 | 先修复 Gateway/browser 接入，不盲重试 |
| 未登录 BOSS直聘 | 提示用户先登录 |
| 验证码/风控 | 暂停自动化，等待用户手动处理 |
| 搜索结果为空 | 建议放宽关键词或城市范围 |
| 某职位详情打不开 | 跳过并继续 |
| Bitable 不存在 | 自动创建或请求用户提供链接 |

---

## 使用前提

1. OpenClaw Gateway 正常运行
2. `profile=user` 已能接入用户浏览器，或已切换到扩展 Relay 方案
3. 浏览器中已登录 BOSS直聘
4. 平台风控较敏感，避免短时间高频打开大量详情页
