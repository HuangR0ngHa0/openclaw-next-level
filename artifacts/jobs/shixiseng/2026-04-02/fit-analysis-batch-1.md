# JD Fit Analysis — Shixiseng Guangzhou C++ Batch 1

## 1) overall verdict

本轮 3 个岗位里，**最值得优先推进的是海量数据的“分布式系统开发工程师（C/C++）”**，其次是蜀俊博凌的“C/C++（实习）”，再次是泰逢游戏的“游戏C++服务端”。

原因不是薪资，而是和用户当前真实背景的**方向一致性 + 可包装性 + 诚实匹配度**。

- **强目标**：海量数据 — 分布式系统开发工程师（C/C++）
- **可投目标**：蜀俊博凌 — C/C++（实习）
- **次优目标 / 可选**：泰逢游戏 — 游戏C++服务端

## 2) strong matches

### A. 海量数据 — 分布式系统开发工程师（C/C++）

**强匹配点：**
- 明确要求 **C/C++ 基础扎实**，与用户主语言方向一致
- 明确要求 **Linux**，与用户 Ubuntu / ROS / Linux 环境高度一致
- 要求 **数据结构、算法、网络、多线程、进程间通信、内存管理**，和用户当前需要强化的工程基础方向一致，且可作为求职主线继续强化
- 强调 **自学能力、逻辑分析、独立解决问题**，用户可以用 DLIO_PRO、无人系统联调、竞赛系统集成经历来支撑
- 广州、本科、实习岗，城市与学历硬条件匹配

### B. 蜀俊博凌 — C/C++（实习）

**强匹配点：**
- 通用 C++ 开发要求，和用户当前背景不冲突
- 要求算法和数据结构基础，用户虽然还需加强，但至少方向一致
- 要求沟通协作和团队合作，这一点和用户的方程式无人系统总监 / 组长经历高度匹配
- 广州、本科、实习岗，硬条件匹配

### C. 泰逢游戏 — 游戏C++服务端

**强匹配点：**
- 要求 C++、Linux、逻辑能力，这些维度用户有部分基础
- 本科、广州、实习岗，硬条件匹配
- 服务端 / 工具开发对 C++ 工程能力有一定通用价值

## 3) partial matches / adjacent evidence

### 蜀俊博凌

- 常见开发工具（VS / Eclipse）并不是核心优势，但这类工具可快速适配，不属于决定性短板

### 泰逢游戏
- 用户没有游戏服务端项目
- 但 C++、Linux、逻辑能力、工具开发意识与该岗存在邻接性

## 4) clear gaps

### 海量数据
- JD 明确写了 **211以上本科大三或研二**，这是明显风险项；用户学校背景不满足该硬标签
- 用户尚无正式实习经历
- 对 **网络、多线程、进程间通信、数据库、内存管理** 的系统化表达还不足

### 蜀俊博凌
- 缺少正式企业软件开发经历
- 缺少能直接证明“高质量 C++ 应用开发” 的成熟商用项目
- 算法 / 数据结构基础虽有关联，但还不算强证据

### 泰逢游戏
- 与用户核心职业目标（机器人 / 自动驾驶 / 系统工程）偏离
- 缺少 Lua、数据库、完整游戏项目经验
- 行业迁移解释成本更高

## 6) risk notes

- 所有岗位都缺少“正式实习经历”这一强证据
- 海量数据岗位存在 **学校门槛风险**，可能在筛选阶段被卡
- 实习僧页面存在验证码 iframe，当前虽未影响可见 JD 提取，但后续批量化稳定性仍需观察
- 当前样本都来自公开可见内容，后续若进入简历投递动作，需要用户人工确认
